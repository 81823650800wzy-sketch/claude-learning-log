认知更新

  • 静态网站从JSON数据驱动到SVG折线图的完整架构链路可行
  • Cloudflare Pages会自动去除.html后缀(308重定向) → URL匹配必须兼容
  • JavaScript const存在暂时性死区(TDZ) → 被调用的函数引用的const必须在调用前声明
  • fetch()相对路径在子页面会解析为错误路径 → 必须使用绝对路径/data/entries.json
  • 事件委托+class切换比innerHTML重建+独立listener高效且稳定
  • wrangler login OAuth在非交互环境需要CLOUDFLARE_API_TOKEN

行为变更

  • 新增learning-log skill → 自动检测+评分+创建节点
  • 移除pcb-designer/kicad → 维护成本>收益，归档到GitHub后清理本地
  • CLAUDE.md更新 → 添加learning-log条目和工作流
  • 建立价值评分体系 → 标签+技能+仓库+行为锚点自动计算节点价值

建立的新习惯

  • 实验→归档→清理闭环: 任何实验性技能先确认卸载路径
  • 自动评分触发: session结束自动评估价值分，≥5自动创建节点
  • 迭代修复模式: v2→v3→v3.5→v4，每次聚焦解决核心bug
  • 事件委托优先: 单一父容器监听器，避免N个独立listener的内存开销

关键教训

  • fetch路径、const TDZ、Cloudflare URL重写——三个"小问题"导致整个站点完全不可交互
  • 前端交互问题99%可以通过浏览器Console看到明确错误 → 先看Console再改代码
  • 持续成本比功能数量更重要 → 用不上的skill每次session都在消耗Token
  • SVG+事件委托+class切换比Canvas+innerHTML更适合数据驱动的静态站点
  • Cloudflare Pages + wrangler deploy是极简且可靠的部署方案
