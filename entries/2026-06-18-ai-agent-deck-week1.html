认知更新

  • BLE 通信需要后台线程运行异步事件循环（bleak），不能阻塞主线程
  • 指数退避重连策略比固定间隔更优雅：2秒 → 4秒 → 8秒 → ... → 30秒上限
  • 心跳保活是 BLE 长连接的必要机制，5秒间隔是经验值
  • GATT 分块发送受限于 MTU，需要应用层协议处理大数据包
  • tkinter 适合轻量级桌面 GUI，500ms 轮询可以实现"实时"状态更新

行为变更

  • BLE 通信模块采用指数退避 + 心跳保活的工业级设计
  • GUI 模块化设计：主窗口、状态面板、控制面板分离
  • UI 组件库独立封装：圆角矩形、渐变、动画、图标全部可复用
  • 命令行参数支持多种运行模式：BLE/CLI/串口/离线

建立的新习惯

  • 嵌入式项目先写 UI 组件库，再组装界面
  • BLE 通信必须有心跳 + 重连机制
  • Python GUI 用模块化设计，主窗口只负责布局

关键教训

  • BLE 心跳是必需品 — 没有心跳的 BLE 连接会在几分钟内被系统断开
  • UI 组件先于界面 — 先做可复用组件（圆角、渐变、图标），再组装页面，代码更干净
  • 指数退避 > 固定间隔 — 重连策略用指数退避，避免网络抖动时疯狂重试
