认知更新

  • Write 工具在 Windows Git Bash 下路径不一致，必须用 bash cat 写文件
  • Theme.MaterialComponents 在部分机型崩溃，Theme.AppCompat 最稳定
  • Button 的 textColor 必须显式设置，否则深色背景不可见
  • ScrollView > LinearLayout 是最可靠的布局结构，避免 FrameLayout 嵌套
  • Camera2/Vibrator 等硬件 API 必须全部 try-catch
  • Gradle 缓存会导致旧代码残留，每次必须 rm -rf build

行为变更

  • 所有教训写入 skill.md，后续生成 APK 时自动遵守
  • create.sh 一键生成项目骨架，build.sh 编译并输出到 F 盘
  • 模板内置 AppUpdater 自更新 + ResourceSync 资源同步模块
  • 删除旧版 apk-builder 和 apk-builder-skill，统一为一个

建立的新习惯

  • 生成 APK 前先读 skill.md 中的"关键教训"
  • 文件写入统一用 bash cat > 而非 Write 工具
  • 编译前必须 rm -rf app/build build
  • APK 统一输出到 F:\ 盘便于传输

关键教训

  • 工具链的隐性差异（路径格式）会造成难以排查的 bug，必须在 skill 中固化
  • "能跑"和"所有设备能跑"是两回事，兼容性需要经验积累
  • Skill 的价值在于把踩坑经验变成可复用的约束
