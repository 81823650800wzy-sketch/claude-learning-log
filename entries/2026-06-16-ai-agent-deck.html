AI Agent Deck - 桌面AI控制终端

ESP32 BLE HID ST7789 Python GUI 硬件开发
◫ AI-Agent-Deck

项目概述

  • 桌面 AI 工作流控制终端 - 一键触发 ChatGPT/Claude/Cursor 等工具
  • 硬件: ESP32-S3 + ST7789 IPS屏 + 6个机械按键 + BLE
  • 软件: BLE HID 键盘 + PC 端管理软件 (Python GUI)
  • 目标: 物理快捷键控制 AI 工具，提升工作效率

技术实现

  • ST7789 驱动: MADCTL=0x40 修正文字镜像，ROW_OFFSET=80
  • BLE HID: ESP-IDF v5.5.4 API，发送 a-f 字母键
  • 按键扫描: 3x2 矩阵，自适应采样率 (5ms/20ms)
  • PC 管理: pynput 监听 + tkinter GUI + 可配置映射
  • 字体渲染: 5x7 点阵字体，支持 1x/2x 缩放

踩坑记录

  • ST7789 文字镜像: 不同模块的 MADCTL 配置不同，需要尝试 MY/MX 位
  • BLE API 变更: ESP-IDF v5.5.4 的 esp_bluedroid_init() 不再需要参数
  • 按键悬空: 未连接的 ADC 引脚会读到随机值
  • F13-F18 键: Windows 不响应这些键，改用普通字母键
  • 采样率: 快速连击会丢失，需要自适应采样

开发成果

  • ✅ Phase 1 基本完成: 屏幕、按键、BLE HID
  • ✅ PC 端管理软件: GUI 界面，可配置按键映射
  • ✅ 一键触发: 按下设备按键自动打开 ChatGPT/Claude/Cursor
  • ⏸️ 待完成: 摇杆、电池管理、UI 菜单、PCB 设计

关键代码片段

  • ST7789 初始化: madctl = 0x40 (MX 位修正镜像)
  • BLE 发送按键: esp_hidd_dev_input_set(dev, 0, 1, buffer, 8)
  • 自适应采样: fast_scan > 0 ? 5ms : 20ms
  • PC 端监听: keyboard.Listener(on_press=callback)

学到的经验

  • 嵌入式开发需要耐心调试硬件接口
  • 参考官方示例代码是解决 API 问题的最佳方式
  • PC-设备通信可以使用特殊键码避免冲突
  • 自适应算法可以显著改善用户体验
  • 文档记录对项目维护非常重要

技能获得

BLE HID 开发 ST7789 屏幕驱动 按键矩阵扫描 Python GUI 开发 ESP-IDF v5.5

行为变化
