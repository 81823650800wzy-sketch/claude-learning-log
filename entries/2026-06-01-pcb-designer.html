认知更新

  • NL→PCB 流水线在"自然语言→模块识别→模板化原理图生成"这一步可行
  • AI 目前无法替代硬件工程师的判断——电路设计需权衡数模混合、EMC、热管理
  • 工具链碎片化：kicad-sch-api (第三方) ≠ kipy (官方) ≠ kcaa MCP (封装层)
  • AI 适合作为 设计助手 而非完全自动设计器

行为变更

  • 移除技能：pcb-designer、kicad — 维护成本高、Token 消耗大
  • 精简 CLAUDE.md：删除所有 PCB/KiCad 条目
  • 卸载数据：53MB 数据完全删除，全部归档到 GitHub

建立的新习惯

  • 实验 → 验证 → 归档 → 清理 的闭环流程
  • 任何实验性技能安装前先确认有卸载路径
  • 归档后立即清理本地文件以节省 Token
  • 主动评估每个技能的维护成本 vs 收益

关键教训

  • 持续成本比功能数量更重要 — 无用 skill 每次会话都在消耗 Token
  • 不要因为理论上能做就默认值得长期维护
  • 工具链选型时优先考虑无头路径，GUI/MCP 作为增强
  • 等 KiCad 11 headless API 成熟后再重启 AI PCB 设计方向
