认知更新

  • NL→PCB 流水线在"自然语言→模块识别→模板化原理图生成"这一步是可行的
  • 但 AI 目前无法替代硬件工程师的判断——电路设计需要权衡数模混合、EMC、热管理等多维因素
  • 当前 pipeline 只能做模板化拼装，无真正的设计智能
  • AI 适合作为 设计助手 而非完全自动设计器
  • 工具链碎片化严重：kicad-sch-api (第三方) ≠ kipy (官方) ≠ kcaa MCP (封装层)
  • easyeda2kicad 依赖 EasyEDA API，部分 LCSC 料号会失效

行为变更

  • 移除技能：pcb-designer (−)、kicad (−) — 维护成本高、收益低、Token 消耗大
  • 精简 CLAUDE.md：删除所有 PCB/KiCad 条目，只保留核心技能
  • 卸载数据：D:\pcb-designer\ (53MB) 完全删除，全部归档到 GitHub

建立的新习惯

  • 实验 → 验证 → 归档 → 清理 的闭环流程
  • 任何实验性技能安装前，先确认有明确的卸载/清理路径
  • 归档到 GitHub 后立即清理本地文件，避免 Token 浪费
  • 主动评估每个技能的维护成本 vs 收益

关键教训

  • 持续成本比功能数量更重要 — 一个用不上的 skill 每次会话都在消耗 Token
  • 不要因为理论上能做就默认值得长期维护
  • 工具链选型时优先考虑无头路径（kicad-cli），GUI/MCP 作为增强
  • 对 AI 辅助 PCB 设计的判断：等 KiCad 11 headless API 成熟后再重启
