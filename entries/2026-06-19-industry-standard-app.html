认知更新

  • 版本单一真相源：__version__ 散落在 3 个文件 → 改为 app/utils/version.py 唯一数据源，其他模块 import。版本发布时只改一处
  • 日志不是 print()：Python logging 模块 + RotatingFileHandler + Qt 信号桥接。控制台+文件+UI 三路输出，生产环境可追溯
  • 崩溃处理器是必需品：sys.excepthook 捕获未处理异常 → 生成崩溃报告 → 显示用户友好对话框。没有它，程序会"消失"得无影无踪
  • 系统托盘 = 专业感：最小化到托盘而非退出，右键菜单提供快捷操作。用户感知从"脚本"升级为"应用"
  • 固件烧录集成：esptool 作为 Python 库调用，烧录前自动释放串口，完成后自动重连。用户体验从"手动命令行"升级为"一键烧录"
  • Inno Setup 安装包：文件关联(.adp)、环境变量PATH、自动卸载旧版本、中英文语言。分发从"解压即用"升级为"安装向导"

行为变更

  • 从"写脚本"到"做产品"：每个功能都要考虑错误处理、日志、用户反馈
  • 清理遗留代码：删除 3 代遗留代码（4680 行），只保留唯一的现代代码库
  • 打包前先复制固件到 firmware/ 目录，确保安装包包含所有必需文件
  • 所有用户可见的错误都要有对话框，不能只 print()

建立的新习惯

  • Python 项目标配：version.py + logger.py + crash_handler.py + pyproject.toml
  • PyQt5 应用标配：系统托盘 + 启动画面 + 关于对话框
  • 硬件配套应用标配：固件烧录集成 + 固件版本管理
  • 发布标配：PyInstaller spec + Inno Setup 脚本 + build.bat
  • 遗留代码清理：删除比保留更安全，git 历史可以回溯

关键教训

  • "能跑"和"能发布"之间隔着：日志、崩溃处理、版本管理、安装包、许可证
  • 清理遗留代码是投资不是浪费 — 4680 行删除换来的是清晰的架构
  • 硬件配套软件的特殊性：固件版本同步、串口占用管理、烧录流程集成
  • 行业标准 = 用户不用看文档就能用，开发者不用猜就能改
